About:

Logcryption is a "encryption" algorithm that uses the Cantor pairing function to condense two characters into one. Although it is techincally encryption, it should only be used to store non-confidential data, and is optimal when the minimum amount of characters are required.

Logcryption is an alphabet of sorts. If you need to store as much data as possible in as little characters as possible, you should use Logcryption.

Pros


Can store lots of data in minimum space
Cool for Easter eggs
Prioritizes availability of more commonly used characters

Cons


No support for higher unicode characters, Common punctuation, Latin Alphabet, and Numeric values are fully supported only
Some fonts cannot read properly
No Security
